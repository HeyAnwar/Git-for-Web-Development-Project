    37. Walk me through Lambda's git flow.
    GIT FLOW
    1. Fork the repo
    2. Clone it and enter the link after you enter - git clone "clone-link" 
    3. Change directory to your newly cloned repo - cd "repo-name"
    4. Create your own branch with - git checkout -b "branch-name"
    5. Prepare your changes to be ready to commit with - git add .
    6. Commit your changes with - git commit -m "message"
    7. Push your committed changes with - git push -u origin "branch-name"
    8. Return to your forked repo on Github and select your branch.
    9. Open a pull request on your branch and make sure you have YOUR branch selected then click create pull request.
